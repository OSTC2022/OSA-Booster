# 캘린더 반복 일정 테스트 체크리스트

> DB 마이그레이션 `supabase/add-calendar-recurrence-v2.sql` 적용 후 테스트하세요.  
> RRULE 확장 로직 자동 검증: `npx tsx scripts/test-recurrence-scenarios.ts`

## 앱에서 생성

| # | 시나리오 | 확인 방법 | 기대 결과 |
|---|----------|-----------|-----------|
| 1 | **매주 목요일 18시** 수업 생성 | 다음 달 캘린더로 이동 | 매주 목요일 18:00 occurrence 표시 (master 1건, DB bulk 없음) |
| 2 | **격주** 수업 생성 | 4~6주 범위 확인 | 정확히 2주 간격 (한 주 건너) |
| 3 | **매월 15일** 수업 생성 | 다음 달로 이동 | 다음 달 15일 occurrence 1건 |

## 수정 범위

| # | 시나리오 | 확인 방법 | 기대 결과 |
|---|----------|-----------|-----------|
| 4 | 반복 중 **하루만 삭제** | 해당 날만 삭제 | 그 날짜만 사라짐, 나머지 occurrence 유지 |
| 5 | 반복 중 **하루만 시간 변경** | 한 occurrence만 20:00 등으로 수정 | 해당 날만 시간 변경, 다른 날은 기존 시간 |
| 6 | **이 날짜 이후** 전체 시간 변경 | 중간 occurrence에서 "이후" 수정 | 이전 일정 = 기존 시간, 이후 = 새 시간 (master 분할) |
| 7 | **전체 반복** 시간/필드 수정 | "전체" 수정 | 모든 occurrence에 반영 |

## 삭제 범위
